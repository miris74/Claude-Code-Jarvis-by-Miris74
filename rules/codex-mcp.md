# Codex MCP サブエージェント運用ガイド

> Codex MCPサーバー接続時のみ適用。
> 単純なタスク（単一ファイル・数行変更、設定編集、ドキュメント更新）はClaude直接処理。

## アーキテクチャ

```
ユーザー → Claude Code（設計・統合）
              ↓
      Codex MCP（実行）
      ├── コード生成・実装
      ├── コードレビュー・品質検査
      ├── リファクタリング
      ├── テスト作成
      ├── デバッグ・根本原因分析
      └── セキュリティ監査
              ↓
      Claude Code（検証・統合・コミット）
```

## MCPツール

| ツール | 目的 |
|------|---------|
| `mcp__codex__codex` | タスク委譲（メイン） |
| `mcp__codex__codex-reply` | sessionIdでセッション継続 |

## 委譲基準

**Codexに委譲**: 20行以上のコード生成、複数ファイル変更、コードレビュー、テスト作成、リファクタリング、根本原因分析、セキュリティスキャン

**Claude直接処理**: 数行変更、設定編集、ドキュメント更新、Git操作、Q&A

## 委譲ルール

1. **プロンプトを自己完結させる** — sessionId未使用時は全情報を含める
2. **7セクション形式**: TASK / EXPECTED OUTCOME / CONTEXT / CONSTRAINTS / MUST DO / MUST NOT DO / OUTPUT FORMAT
3. **レビューはCodexに委譲** — Claudeは最終判断のみ
4. **セッション継続を活用** — 関連タスクはsessionIdで維持
5. **エラー時はリトライ** — ステートレス: 元タスク+エラー詳細、セッション: codex-reply
6. **出力をそのまま渡さない** — 必ず検証・統合・整形してから報告

## セキュリティ

- APIキー・秘密情報をプロンプトに含めない
- 生成コードは必ずセキュリティレビュー
